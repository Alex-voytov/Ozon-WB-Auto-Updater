# Ozon/WB Auto Updater

Десктопное приложение с графическим интерфейсом (Python + Tkinter), которое автоматически обновляет названия и описания товаров на **Ozon** и **Wildberries**, генерируя SEO-тексты через **Claude** или **Gemini** на основе реальных ключевых слов и анализа конкурентов.

## Возможности

- Генерация SEO-описаний для карточек Ozon и WB через Claude или Gemini, с проверкой текста на орфографию, пунктуацию и логическую согласованность.
- Сбор ключевых слов из реальной аналитики: поисковые запросы покупателей по SKU (Ozon Seller API), позиции товара по nmID (MPStats) — с запасным вариантом через веб-поиск Claude, если аналитики ещё нет (новый товар).
- Анализ карточек конкурентов: сначала через публичный поиск маркетплейса, при недоступности (антибот-защита) — через веб-поиск Claude.
- Предпросмотр «было / стало» перед применением каждого изменения и ручной выбор конкретных товаров для обработки.
- Сравнение и синхронизация цен между Ozon, Wildberries и MPStats.
- Копирование названия и описания с Ozon на WB для товаров, сопоставленных по артикулу (offer_id = vendorCode).
- Постраничная работа с большими каталогами, ретраи с бэкоффом и корректная обработка лимитов API (429/500).
- Логирование всего процесса в консоль приложения и файл `marketplace_manager.log`.

## Требования

- Python **3.10** или выше
- Tkinter (обычно идёт в комплекте с Python; на Linux может потребоваться `sudo apt install python3-tk`)
- Аккаунт продавца на Ozon и/или Wildberries с доступом к API
- Ключ API Anthropic (Claude) и/или Google (Gemini) для генерации описаний

## Установка и запуск

### 1. Клонируйте репозиторий

    git clone https://github.com/Alex-voytov/Ozon-WB-Auto-Updater.git
    cd Ozon-WB-Auto-Updater

### 2. Создайте и активируйте виртуальное окружение

    python -m venv venv
    source venv/bin/activate      # Linux / MacOS
    # или
    venv\Scripts\activate         # Windows

### 3. Установите зависимости

    pip install -r requirements.txt

### 4. Настройте параметры

Скопируйте `config.example.json` в `config.json` и заполните своими ключами (см. раздел [Конфигурация](#конфигурация)).

    cp config.example.json config.json

### 5. Запустите приложение

    python ozon_auto_updater.py

## Использование

1. Откройте вкладку **Настройки** и убедитесь, что ключи Ozon/WB/AI подтянулись из `config.json`; при необходимости проверьте подключение кнопками «Проверить».
2. На вкладке **Товары** нажмите «Загрузить товары» — приложение сопоставит карточки Ozon и WB по артикулу и покажет, где есть расхождения.
3. На вкладке обновления описаний (Ozon или WB) нажмите «Запустить обновление описаний» — при включённом предпросмотре перед каждым изменением будет показано «было / стало» для подтверждения.
4. На вкладке **Цены** сравните и при необходимости синхронизируйте цены между площадками.
5. Чтобы перенести название и описание с Ozon на WB для уже совпадающих по артикулу товаров, используйте кнопку «Скопировать название и описание Ozon → WB» на вкладке товаров WB — можно выбрать конкретные позиции вручную.
6. Весь ход выполнения логируется в консоли соответствующей вкладки и дублируется в файл `marketplace_manager.log`.

## Конфигурация

Файл `config.json` (создаётся на основе `config.example.json`) содержит:

- **`ozon.client_id`** / **`ozon.api_key`** — данные Ozon Seller API (личный кабинет продавца → Настройки → API-ключи).
- **`wb.api_key`** — токен Wildberries Content API.
- **`mpstats.token`** — токен [mpstats.io](https://mpstats.io) (опционально, для аналитики позиций WB).
- **`performance.enabled`** / **`performance.client_id`** / **`performance.client_secret`** — доступ к Ozon Performance API (опционально).
- **`ai.provider`** — генератор описаний по умолчанию: `claude` или `gemini`.
- **`ai.anthropic_api_key`** / **`ai.model`** — ключ и модель Claude.
- **`ai.gemini_api_key`** / **`ai.gemini_model`** — ключ и модель Gemini (опционально).
- **`update.days_back`** — глубина анализа поисковых запросов покупателей, дней.
- **`update.top_keywords`** — сколько ключевых слов брать для генерации описания.
- **`update.target_offer_ids`** — список конкретных offer_id для точечной обработки (пусто — обрабатываются все).

**`config.json` содержит секреты и не должен попадать в репозиторий** — он уже добавлен в `.gitignore`.

## Зависимости

Основные библиотеки, используемые в проекте:

- `anthropic >= 0.113.0` — генерация описаний через Claude и веб-поиск конкурентов/ключевых слов.
- `requests >= 2.31.0` — обращения к API Ozon, Wildberries и MPStats.
- `schedule >= 1.2.0` — периодический запуск задач обновления.
- `google-genai >= 0.3.0` — генерация описаний через Gemini (опционально, если выбран этот провайдер).

> **Примечание:** графический интерфейс использует `tkinter` из стандартной библиотеки Python — устанавливать отдельно не нужно (кроме некоторых Linux-дистрибутивов).
